# Contributing

Contributions are welcome.

## Guidelines

- Keep the skill generic and portable across AI CLI tools.
- Prefer semantic rules over long keyword lists, but keep important routing hints in the description.
- Avoid adding task-domain-specific examples to `SKILL.md` unless they clarify a general rule.
- Keep task record structure concise enough for long-term use.
- Preserve multilingual behavior and exact code/command/path text.

## Editing Checklist

- Update `SKILL.md` for behavior changes.
- Update `README.md` and `README.zh-CN.md` if usage changes.
- Update `CHANGELOG.md` for notable changes.
- Restart the target CLI when testing skill-loading behavior.
